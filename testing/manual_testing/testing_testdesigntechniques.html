Test Design Techniques

There are basically two types of Testing Techniques.

  • White box Testing.
  • Black box Testing.

White box Testing :

To check the Internal logic of each program structure. It is also called structural testing or open box or glass box or a clear box.

White box testing Techniques are

  • Statement Testing
  • Branch/Decision Testing
  • Data Flow Testing
  • Branch Condition Testing
  • Branch Condition Combination Testing
  • Modified Condition Decision Testing
  • LCSJ(Linear Code Sequence and Jump)

Statement Testing: In Component/unit testing, statement coverage is the assessment of the percentage of executable statements that have been exercised by a test suite.

Statement: A statement is an entity in a programming language, which is typically the smallest unit of execution.

Decision Testing & Coverage: A Decision is a program point at which the control flow has two or more alternative routes. Decision coverage is the percentage of the decision outcomes that have been exercised by a test suite.

Data flow testing: Data flow testing is a family of test strategies based on selecting paths through the program's control flow in order to explore sequences of events related to the status of variables or data objects. It focuses on the points at which variables receive values and the points at which these values are used.

Branch Condition Testing: Branch Condition Testing requires a model of the source code which identifies decisions and the individual Boolean operands within the decision conditions. A decision is an executable statement which may transfer control to another statement depending upon the logic of the decision statement.

Modified Condition Decision Testing: Modified Condition enhances the condition/decision coverage criteria that each condition is shown to independently affect the outcome of the decision. This kind of testing is performed on a mission-critical application which might lead to death, injury or monetary loss. Designing Modified Condition Coverage or Decision Coverage requires more thoughtful selection of test cases which is carried out on a stand-alone module or integrated components.

LCSJ(Linear Code Sequence and Jump): LCSAJ stands for Linear Code Sequence and Jump, a white box testing technique to identify the code coverage, which begins at the start of the program or branch and ends at the end of the program or the branch. LCSAJ consists of testing and is equivalent to statement coverage and branch coverage.

Black box testing: To check the External functionality of the application. It is also called as Functionality testing or Behaviour or Requirement based or closed box testing.

Black box testing Techniques are as follows:

  • Equivalence Class Partitioning
  • Boundary Value Analysis
  • Decision Table
  • State Transition Testing
  • Syntax Testing
  • Random Testing

Equivalence Class Partitioning: It is the process of reducing a huge set of possible test cases into a much smaller, but still equally effective set. It means that it is tested with both valid and invalid data.

Applicability: Most suites to systems in which much of the input data takes on values within ranges or within sets.

Advantages: It enables a tester to select a subset of test inputs with a high probability of detecting a defect.

Disadvantages: It makes the assumption that the data in the same equivalence class is processed in the same way by the system.

Boundary Value Analysis: It is the act of dividing the given input of set by a relation into groups or classes which are treated the same by the module or which should produce the same result

It is technique to test the range of the inputs in terms of On boundary, Below boundary and above boundary as follows Min, Min-1, Min+1, Max, Max-1, Max+1.

Applicability: Typically be suited to systems in which much of the input data takes on ranges of values or within sets.

Decision Table: These are used to describe and analyze problems that contain procedural decision situations that are characterized by one or more conditions.

Applicability: It can be used whenever the system must implement complex business rules.

Advantages: Enables us to look at and consider “dependence”, “impossible” and not relevant situations and eliminate some test cases.

State Transition Testing: State transition diagrams are an excellent tool to capture certain types of system requirements and to document internal system design.

State: It is a condition in which a system is waiting for one or more events. It is generally represented by circles.

Transition: Represent a change from one state to another caused by an event. In the diagrams, transitions are represented by arrows.

Event: It is something that causes the system to change state. In diagrams, events are represented by labels on a transition.

Applicability: The application can be characterized by a set of states.

Advantages: The state transition diagram gives a pictorial presentation of the states and the entire application flow.
